## 1. Базовая информация

### Термины

- **ККТ.** Касса
- **ФФД.** Формат фискальных данных. Есть 1.05 (без маркировки и НДС22%, устарел, но разрешен) и 1.2. По сути описание всех полей xml, которые можно передаввать в чеке.
- **ФН.** Фискальный накопитель
- **ЗН ККТ** - заводской номер ККТ, смотреть в тест-драйвере
- **РН ККТ** - регистрационный номер ККТ, выдается в налоговой при регистрации ККТ, смотреть в ЛК ИФНС

### 1.1. ККТЛаб, Посцентр и Торгбаланс

После распада ГК «ШТРИХ-М» образовались **три независимые компании**, каждая обслуживает кассы. Каждая выпускает свои драйвера, прошивки и лицензии. Лицензии платные у каждого производителя и устанавливаются только на прошивку этого производителя. У всех все одно и то же с одним нюансом: лицензии одного производителя не будут работать с прошивками другого производителя. Драйвера можно использовать от любого производителя, но желательно использовать драйвера от того же производителя, что и прошивка.

### 1.2. Как устроена память ККТ: загрузчик и прошивка

- **Загрузчик** + **Прошивка** — две части ПО ККТ.

- При включении ККТ сначала загружается загрузчик, проверяет прошивку и если все ок, то загружает ее, иначе сообщает об ошибках сигналами.
- Если есть проблемы с прошивкой, то загрузчик позволит загрузить новую через утилиту DFU
- Версия загрузчика читается в тест-драйвере:
  ```
  09. Параметры устройства → Загрузчик → Прочитать версию загрузчика
  ```
- Для DFU нужна версия загрузчика **≥131** (у большинства On-Line уже 133).

Если установка загрузчика привела к крипичу - нужно либо шить новый через SD-карту, либо через программатор.

Так как у ШТРИХ-ON-LINE не распаян разъем для SD-карты, при неудачном обновлении загрузчика придется ехать в ЦТО.

### 1.3. Версии прошивок

У разных производителей свои серии прошивок:

**KKTLAB и Посцентр — серия C:**

| Версия  | ФФД  | Особенности                                         |
| ------- | ---- | --------------------------------------------------- |
| **C.1** | 1.05 | Без новых ставок НДС и маркировки (нет новых тегов) |
| **C.3** | 1.2  | С поддержкой маркировки (новые теги, другой формат) |

**Торгбаланс — серия T**:

| Версия  | ФФД  | Особенности             |
| ------- | ---- | ----------------------- |
| **T.1** | 1.05 | Без маркировки          |
| **T.3** | 1.2  | С поддержкой маркировки |

**Варианты прошивок для ШТРИХ-ON-LINE у Посцентр и ККТЛаб:**

- **С ключами** (`upd_app_…bin`) — для касс, где в таблице 23, поле 11 (UIN) указан **номер** (например, `400...`).
- **Без ключей** (`upd_app_for_old_frs…bin`) — для устройств с **прочерками** в UIN.

У Торгбаланса таких заморочек нет.

**Частые вопросы:**

- **Можно ли переходить с C.1 на C.3 или Т.3?** Да, можно переходить на что угодно и в любом порядке. Можно и обратно, если не требуется маркировка и новые сценарии, завязанные на ФФД 1.2.
- **Можно ли оставаться на C.1?** Да, если не нужна маркировка и новые сценарии, завязанные на ФФД 1.2.
- **Можно ли переходить с C-серии на T-серию?** Да, обратно тоже можно.

**Текущая версия прошивки - где смотреть:**

```
01. Состояние → Длинный запрос
Строки: "Версия ПО", "Дата ПО"
```

### 1.5. DFU-режим

Информационный разде, можно пропустить.

DFU - режим загрузки обновлений.

В тест-драйвере есть раздел Сервис - Прошивка, если при прошивке получаем ошибку перехода в DFU-режим, то можно перейти в него вручную.

**Два способа перевода в DFU:**

1. **Аппаратный (джампер, для тяжелых случаев, не рекомендуется):**
   - Вскрыть корпус, переставить джампер на плате.

2. **Программный (команда FE ED, рекомендуется):**

   ```
   12. Прочее → Команда → HEX-команда: FE ED 00 00 00 00 → Выполнить
   ```

   - Касса перезагружается.
   - В диспетчере устройств появляется **DFU-устройство**.
   - В течение 30 секунд запустить DFU Utility.

В дальнейшем касса будет нормально шиться из тест-драйвера

### 1.6. SD-карты

- На большинстве Штрих-On-Line **слот microSD не распаян** (есть только контактная площадка).
- Прошивка с SD-карты **невозможна** (да и не нужна)

### 1.7. Лицензии

**Лицензия:**

- Привязка: к заводскому номеру ККТ
- Назначение: право использовать новые возможности (НДС 22%, новые ФФД)

**Где посмотреть:**

```
Тест-драйвер → Сервис → Лицензии ККТ
```

**Как записать:**

```
Тест-драйвер → Сервис → Лицензии ККТ → Указать файл/код от поставщика → Записать в ККТ
```

**Работа без лицензии:**
Без лицензии новые прошвки не работают совсем.  
Если нет нужных лицензий, то касса при включении и попытке открыть смену будет печатать ошибку на ленте.

## 2. Прошивка

Если установка прошивки привела к кирпичу - ничего страшного, можно залить новую

### Подготовка

#### Проверка версии загрузчика

```
09. Параметры устройства → Загрузчик → Прочитать версию загрузчика
```

- **Версия <131** — DFU может не поддерживаться, нужен XMODEM.
- **Версия ≥131** (например, 133) — DFU работает.

#### Проверка UIN (определение типа прошивки длч Посцентра и ККТЛаб)

```
Таблицы → Таблица 23 "Удалённый мониторинг и администрирование" → Поле 11: Uin (ro)
```

**Варианты:**

| Значение UIN                 | Нужная прошивка | Файл                       |
| ---------------------------- | --------------- | -------------------------- |
| **Прочерки (`---`)**         | Без ключей      | `upd_app_for_old_frs_…bin` |
| **Цифры (например, `400…`)** | С ключами       | `upd_app_…bin`             |

#### Сохранение таблиц

```
Таблицы → Импорт/Экспорт → Экспорт всех таблиц → Сохранить файл (например, tables_backup.bin)
```

Если после прошивки что-то слетит в настройках, можно их восстановить.

---

### Прошивка Торгбаланс: через утилиту обновления

---

- Запустить утилиту [fwupdate_1.0.1.54_x64.exe](Прошивки\torgbalance\fwupdate_1.0.1.54_x64.exe)

- если после прошивки ошибка 116 и ККТ мигает красным - выполнить [обнуление и установку даты и времени](#код-116-ошибка-озу)

## 3. Замена ФН

**ВНИМАНИЕ! НИ НА ОДНОМ ЭТАПЕ НЕ НУЖНО ВВОДИТЬ ДАННЫЕ ВРУЧНУЮ!**
**ВСЕ МОЖНО СОХРАНИТЬ ИЛИ ЗАПОЛНИТЬ!**
**ОТЧЕТЫ О ЗАКРЫТИИ И РЕГИСТАРЦИИ ОБЯЗАТЕЛЬНО СОХРАНИТЬ ЧЕРЕЗ ТЕСТ-ДРАЙВЕР!**

Новый ФН можно проверить заранее здесь: https://kkt-online.nalog.ru/#check-fn

[Инструкция по замене ФН](https://tensor.ru/cto/zamena_fn_shtrih)

[инструкция по перерегистрации на сайти ИФНС](https://saby.ru/help/ofd/setting/rereg/fns)

#### Закрытие старого ФН

```
1. Закрываем смену (Z-отчёт)
2. Делаем архив ФН (02. ФН сервис → Архив ФН → Прочитать и сохранить архив ФН в файл)
3. Дожидаемся отправки всех ФД в ОФД
4. Закрываем ФН (02. ФН сервис → Состояние ФН → Начать закр. фиск. режима → Закрыть фискальный режим)
5. Сохраняем данные закрытия в файл: (02. ФН сервис → Данные → Прочитать последний документ → Сохранить в файл)
```

#### Физическая замена ФН

```
1. Выключаем кассу, вынимаем питание
2. Откручиваем 4 винта снизу (крышка отсека ФН)
3. Вынимаем старый ФН (аккуратно, не повреждаем разъём)
4. Вставляем новый ФН до упора (по ключу разъёма)
5. Закрываем крышку, закручиваем винты
6. Включаем питание
```

#### Регистрация нового ФН

```
2.  ФН сервис → Проверка ФН
    Статус должен быть: "Новый (не активирован)"

3.  ФН сервис → Фискализация ФН →
Данные заполнятся, проверить:
- ИНН организации
- Наименование организации / ИП
- Адрес расчётов
- Режим работы (автономный / с ОФД)
- Признак ККТ (интернет-ымагазин и т.д.)

4. Нажать "Начать отчет о регистрации ККТ" → "Сформировать отчет о регистрации"
5. Сохраняем данные регистрации в файл: (02. ФН сервис → Данные → Прочитать последний документ → Сохранить в файл)
```

#### Регистрация в ФНС

```
1. Заходим в личный кабинет ФНС (lkk.nalog.ru)
2. Перерегистрируем ККТ с новым номером ФН: выбрать ккт, ввести дату и номер отчетов о закрытии и регистрации
3. Дожидаемся окончания обработки
```

## 5. Проблемы и решения

### Код 116 Ошибка ОЗУ

#### Причины:

| Причина                        | Решение                                        |
| ------------------------------ | ---------------------------------------------- |
| **Нужно техобнуление**         | Выполняем технологическое обнуление (см. ниже) |
| **Неправильный файл прошивки** | Шьем другую прошивку                           |

#### Алгоритм устранения:

**Технологическое обнуление**

```
10. Сервис → Вкладка "Обнуление" → Технологическое обнуление
Дожидаемся: (0) Ошибок нет
```

**Установка даты и времени**

```
10. Сервис → Вкладка "Дата и время"
Устанавливаем текущие дату и время
```

**Перезагрузка**

```
Выключаем кассу (тумблер или питание)
Ждём 5 секунд
Включаем снова
```

#### Если ошибка остаётся:

1. Убеждаемся, что используем **правильный файл** (с ключами / без ключей).
2. Проверяем **физическое подключение** ФН (шлейфы, контакты).

#### Проверка успешной прошивки

```
1. Состояние → Длинный запрос

```

10. Сервис → Обнуление → Технологическое обнуление

- Прочие параметры

### Прошивка через DFU Utility (если не получается через тест-драйвер)

Переводим ККТ в редим DFU:

```
Тест-драйвер → 12. Прочее → Вкладка "Команда"
Поле "Команда HEX": FE ED 00 00 00 00
Нажимаем "Передать"
```

**Что происходит:**

- Касса перезагружается.
- Загорается **красный индикатор**.
- В **Диспетчере устройств Windows** появляется **DFU-устройство** (раздел "Универсальная последовательная шина USB" или "Другие устройства").
- После перевода в DFU режим касса остаётся в этом состоянии **30 секунд**. Успеваем запустить DFU Utility.

---

Прошиваем через утлиту DFU:

```
Копируем файл прошивки в папку DFU и переименовываем в `upd_app.bin`
В папке DFU находим и запускаем:
"Прошивка через DFU.bat"
```
